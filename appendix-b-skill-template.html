模板与规范

官方 SKILL.md 规范解读，以及本书使用的6字段实践增强版模板

Background · 背景
SKILL.md 是什么

2025年12月，Agent Skills 规范由 agentskills.io 发布，随即被 OpenAI Codex、GitHub Copilot、VS Code、Cursor 等50多个主流 AI 工具采纳为跨平台标准。它的核心思路是：用一个纯文本文件（SKILL.md）来描述一项可复用的 AI 技能，让任何支持该标准的工具都能理解和调用它。

就像 README.md 是每个代码仓库的自我介绍，SKILL.md 是一项 AI 技能的自我介绍。你写好这张"技能身份证"，就能把你积累的经验以标准格式分享给团队，或者迁移到不同的 AI 工具里继续使用。

Official vs Extended · 规范层级
官方必填字段 vs 工具扩展字段

理解这个区别很重要：官方规范极简，只要求两个字段。其余字段都是各工具在官方基础上扩展的，不同工具支持的字段可能不同。

✅ 官方规范 agentskills.io
官方必须遵守的最小集合
name: 会议纪要生成器 description: 将会议录音/文字 转化为结构化纪要 # 仅这两个字段是官方规范 # 任何支持 SKILL.md 的工具 # 都能读懂
🌐 跨平台兼容：所有支持 Agent Skills 规范的工具都必须支持这两个字段。
🔧 工具扩展字段 第三方扩展
Claude Code 等工具在官方基础上扩展
name: 会议纪要生成器 description: 将会议录音/文字 转化为结构化纪要 effort: medium model: claude-opus-4 context: fork # effort/model/context 是 # Claude Code 等工具的扩展字段 # 其他工具可能不支持
⚠️ 注意：这些扩展字段仅在支持它们的特定工具中生效，迁移到其他工具时可能被忽略。
Field Reference · 字段说明
所有字段一览

下表汇总了官方必填字段、常见扩展字段，以及本书实践版中增加的白领工作专属字段。

字段名 类型 含义与用途 来源
name必填 字符串 技能的唯一名称。简洁有辨识度，建议中文命名方便团队检索。 官方规范
description必填 多行文本 详细描述这项技能做什么、怎么用、适用场景。AI 通过读这段文字来理解该技能。写得越清晰，调用越准确。 官方规范
effort扩展 low / medium / high 预期任务耗时/复杂度。帮助工具决定是否需要更强的模型或更长的超时时间。 Claude Code 扩展
model扩展 字符串 指定调用哪个 AI 模型。通常复杂分析用大模型，简单格式化用轻量模型节省成本。 Claude Code 扩展
context扩展 fork / ... fork 表示为每次调用创建独立的上下文，避免多任务相互干扰，支持并行批量处理。 Claude Code 扩展
triggers书中扩展 列表 什么情况下应该调用这项技能？帮助 AI 自动识别合适的使用时机。 本书实践版
output_format书中扩展 字符串 输出格式说明，如 Markdown、表格、JSON。让每次调用输出保持一致。 本书实践版
examples书中扩展 多行文本 2-3个输入→输出示例，帮助 AI 理解期望效果。类似 Few-Shot Prompting 的效果。 本书实践版
Blank Template · 空白模板
本书6字段实践增强版（可打印填写）

复制下方结构，填入你自己的技能信息，就是一张属于你的 Skill 卡片。前两个字段符合官方规范，其余四个字段是本书根据白领工作场景设计的实践增强，在支持扩展字段的工具中能获得更精准的调用效果。

📋 我的 Skill 卡片
name: 官方必填
为这项技能起一个名字，例如：周报速写器
description: 官方必填
详细描述这项技能的功能、使用方法和适用场景。这是 AI 理解该技能的核心依据，写得越清晰效果越好。
effort: 工具扩展
low / medium / high（任务复杂程度）
triggers: 书中扩展
描述在什么情况下应该调用这项技能，例如：当用户需要整理会议内容时
output_format: 书中扩展
描述输出的格式要求，例如：Markdown 格式，包含摘要、待办事项、决议三部分
examples: 书中扩展
提供1-2个输入→输出示例，帮助 AI 理解期望效果
Filled Example · 填写示例
示例：「周报速写器」Skill 卡片

以下是一张填写完整的实际 Skill 卡片，你可以直接修改成自己的内容。

SKILL.md — 完整示例
--- name: 周报速写器 description: 根据用户本周的工作流水账， 自动生成结构化周报。 输出包含：本周完成事项、 遇到的问题与解决方案、 下周计划三个部分。 语言简洁专业，适合直接发给上级。 effort: low triggers: 当用户提供本周工作记录， 需要整理成正式周报时 output_format: Markdown 格式 ## 本周完成 ## 问题与解决 ## 下周计划 examples: 输入：周一开了产品评审， 周三改了3个bug， 周五发布了v2.1版本 输出： ## 本周完成 - 主导产品评审，输出评审意见3条 - 修复线上缺陷3项，完成 v2.1 版本发布 ---
📁 如何使用：把填写好的内容保存为 SKILL.md，放在你的工作目录里，或者上传到支持 Agent Skills 规范的工具（如 Claude Code）。工具会自动读取这张卡片，让你用技能名称快速调用对应功能。
